Desarrollando un sistema de citas médicas: mi experiencia

Una de las experiencias más enriquecedoras de mi formación como estudiante de Ingeniería de Sistemas fue desarrollar un sistema de gestión de citas médicas para una clínica. Este proyecto fue parte de un trabajo académico, pero lo abordé como si fuera para un cliente real.

¿Cuál era el problema?

La clínica necesitaba organizar sus procesos de atención: agendamiento de citas, asignación de doctores y control de horarios. Todo se hacía manualmente, lo que generaba errores, sobrecarga de personal y pérdida de tiempo.

Fase de análisis

Comencé levantando los requerimientos funcionales y no funcionales. Luego utilicé diagramas UML como el de casos de uso y el de actividades para entender la lógica del sistema. Herramientas como Lucidchart fueron clave.

“Diseñar antes de programar me permitió tener claridad sobre cada parte del sistema, evitando errores futuros.”

Diseño y base de datos

Utilicé PostgreSQL para el diseño de la base de datos, asegurándome de normalizar las tablas correctamente. El sistema incluía módulos para pacientes, médicos, horarios y citas.

Interfaces y prototipo

El prototipo fue diseñado con HTML, CSS y JavaScript. También planteé su versión final en Figma para tener una idea clara del diseño responsivo y amigable para el usuario.

Lecciones aprendidas

  • Escuchar bien al usuario final es clave.
  • Modelar antes de codificar ahorra tiempo y errores.
  • Documentar cada paso facilita futuras mejoras.

Este proyecto me motivó aún más a seguir mi camino hacia el desarrollo full stack. Pronto espero integrar bases de datos, backend y frontend en una misma aplicación completa y funcional.
